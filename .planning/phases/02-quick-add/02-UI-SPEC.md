## Spacing Scale

Declared values (all multiples of 4px). Already defined as CSS custom properties in `src/app.css`:

| Token | CSS var | Value | Usage in Phase 2 |
|-------|---------|-------|-----------------|
| xs | `--space-xs` | 4px | Gap between emoji and label on category tile; inline icon gaps |
| sm | `--space-sm` | 8px | Gap between numpad rows; gap between expense list items |
| md | `--space-md` | 16px | Horizontal page padding; numpad key inner padding |
| lg | `--space-lg` | 24px | Section separation (numpad ↔ category grid, category grid ↔ today list) |
| xl | `--space-xl` | 32px | Top padding of page; bottom safe-area offset |
| 2xl | `--space-2xl` | 48px | Bottom sheet slide-up clearance above safe area |
| 3xl | `--space-3xl` | 64px | Not used in Phase 2 |

Exceptions:
- Numpad keys: minimum tap target 56px height (taller than `--tap-min` 44px because numpad is the primary interaction surface; calculator feel requires generous key height).
- Category tiles: minimum 64px height to accommodate stacked emoji + label and meet WCAG 2.5.5 (44px minimum; 64px chosen for comfort on iOS).
- `--tap-min: 44px` (already defined) — used for secondary actions: "Skip", "Save note", back button, gear icon.
- `--button-height: 48px` (already defined) — used for "Save note" CTA inside the bottom sheet.

## Color

All values are already in `src/app.css`. No new colors are introduced in Phase 2.

| Role | CSS var | Hex | Usage |
|------|---------|-----|-------|
| Dominant (60%) | `--color-bg` | `#F7F3EB` | App background, full-page background |
| Secondary (30%) | `--color-surface` | `#E7DED0` | Expense list rows, bottom sheet background, category tile background, edit form card |
| Accent (10%) | `--color-accent` | `#176B5D` | Reserved-for list below |
| Accent foreground | `--color-accent-foreground` | `#FFFFFF` | Text on accent-filled elements |

| Destructive | `--color-destructive` | `#B9382F` | Delete action in edit form only |
| Destructive foreground | `--color-destructive-foreground` | `#FFFFFF` | Text on destructive-filled elements |
| Border | `--color-border` (use `--color-surface`) | `#E7DED0` | Card borders, input outlines, dividers |

Note: `--color-border` and `--color-text` are referenced in Phase 1 components with fallback values (`var(--color-border, #E7DED0)`, `var(--color-text, #1A1A1A)`). **Accent (`#176B5D`) is reserved for these specific elements only:**
1. The amount display text when a non-zero amount is entered (active state)
2. The "Save note" button fill in the bottom sheet
3. The "Save changes" button fill in the edit form (`/expenses/[id]/edit`)
4. The gear icon active/pressed state
5. Category tile pressed highlight border (see Interaction section)

Accent is NOT applied to: numpad keys, the category tile background (normal state), expense list rows, or any muted/secondary text.

## Screen Specifications

### Screen 1: Home (`/`) — Single scrollable page

**Layout (top to bottom, single column, `max-w-sm` centered, `px-4`):**

1. **Install guidance banner** — `InstallGuidanceBanner.svelte` (existing component, keep at top). Hidden in standalone mode.
2. **Gear icon** — top-right, `position: absolute` or flex row with `justify-between`. Uses Unicode ⚙ character at 20px, `--color-muted`. Touch target 44px × 44px. Opens dropdown (see Gear Menu spec).
3. **Amount display** — centered, 48px semibold. Shows "0" in `--color-muted` when empty. Shows IDR-formatted amount (e.g. "54.000") in `--color-foreground` when digits entered. No "Rp" prefix or "IDR" in the display. No currency symbol — raw number with dot separators only.
4. **Numpad grid** — 4 rows × 3 columns of `<button>` elements:
   - Row 1: 7, 8, 9
   - Row 2: 4, 5, 6
   - Row 3: 1, 2, 3
   - Row 4: "000", 0, ⌫ (backspace)
   - Key size: full column width (using CSS grid, `grid-cols-3`, `gap-2`), height 56px minimum.
   - Key style: background `--color-surface`, border-radius 12px, text 20px semibold, `--color-foreground`.
   - Active/pressed state: background darkens to `#D4C9B8` (10% darker than `--color-surface`) immediately on `touchstart`, released on `touchend`. Use `touch-action: manipulation` to eliminate 300ms tap delay.
   - Backspace key (⌫): same size and style as digit keys.
   - "000" key: same size and style as digit keys — it is a first-class key, not a secondary shortcut.
   - No decimal key (IDR has no sub-rupiah amounts — `REQUIREMENTS.md` explicit exclusion).
5. **Category grid** — below numpad, `gap-lg (24px)` vertical separation:
   - 3-column grid (`grid-cols-3`, `gap-2`), 3 rows: 3-3-1 layout.
   - Row 1: 🍜 Food, 🚗 Transport, 🎬 Entertainment
   - Row 2: 💡 Utilities, 🛍️ Shopping, ❤️ Health
   - Row 3: ✅ Others (spans all 3 columns OR left-aligned in col 1 — use `col-span-1` flush left to avoid an artificially wide single tile)
   - Tile style: background `--color-surface`, border-radius 12px, padding 12px 8px, height min 64px. Flex column: emoji (20px) above label (14px, `--color-foreground`).
   - Tile normal state: no border, no color coding.
   - Tile pressed/active state (500ms debounce window per D-14): border `2px solid var(--color-accent)`, background slightly darkens to `#D4C9B8`. State is applied immediately on tap, released after 500ms (or after the note sheet appears, whichever comes first). Use `pointer-events: none` on the tile during the 500ms window to prevent double-tap (implements INPUT-06 debounce visually).
   - No other animation or color change on tiles.
6. **Today expense list** — below category grid, `gap-lg (24px)` vertical separation. Section header: "Today" at 20px semibold.
   - Each row: `min-h-[48px]`, flex row `justify-between items-center`. Left: category emoji + name (16px semibold) + optional note in muted (14px). Right: amount in IDR full format with "Rp " prefix (16px semibold) + date/time in muted (14px).
   - Row background: transparent. Divider: 1px `--color-surface` between rows. No card wrapping per row — flat list.
   - Tapping any row navigates to `/expenses/[id]/edit` (D-06).
7. **"View all history →" link** — bottom of today list (or below empty state). 14px, `--color-muted`. Navigates to `/expenses`.
8. **Empty state (no expenses today):** centered inside a `--color-surface` rounded card (12px radius, `p-4`). Heading: "No expenses yet today" (16px semibold). Body: "Tap a category above to log your first one." (14px, `--color-muted`).

### Screen 2: Note Entry Bottom Sheet (slides up after save)

**Trigger:** Fires immediately after a category tile tap saves the expense (D-07).

**Animation:** Slides up from bottom over 250ms using `transform: translateY(0)` from `translateY(100%)`. Easing: `cubic-bezier(0.25, 0.46, 0.45, 0.94)` (ease-out). The sheet covers the numpad area — it does NOT cover the full screen. Sheet height: ~50% of viewport (approximately 320px on a 390px iPhone).

**Sheet layout (top to bottom, `px-4 py-6`):**
1. Handle bar: 32px wide, 4px tall, `--color-surface` (darker tone), centered, `mb-4`. (Decorative, no drag-to-dismiss; dismiss is explicit via buttons only per D-08.)
2. Saved expense summary: "[emoji] [Category] · [amount with Rp prefix]" — 20px semibold, `--color-foreground`. Example: "🍜 Food · Rp 54.000". Purpose: confirms what was saved (per `<specifics>` note).
3. Note label: "Add a note (optional)" — 14px, `--color-muted`, `mt-4`.
4. Text input: full width, `min-h-[48px]`, border 1px `--color-surface`, border-radius 8px, padding `12px`, 16px regular. Placeholder: "e.g. lunch with team". `autofocus` attribute set on mount (system keyboard opens for text input — this is intentional; the sheet is a post-save overlay, not the quick-add flow).
5. Two buttons side by side (`gap-2`, `mt-4`):
   - "Skip" — secondary style: border 1px `--color-surface`, background transparent, 16px semibold, `--color-muted`, `min-h-[48px]`, border-radius 8px, flex-1.
   - "Save note" — primary style: background `--color-accent`, text `--color-accent-foreground`, 16px semibold, `min-h-[48px]`, border-radius 8px, flex-1.
6. Sheet background: `--color-bg` with `border-top-left-radius: 20px; border-top-right-radius: 20px; box-shadow: 0 -4px 24px rgba(0,0,0,0.12)`.

**Dismiss behavior:** Both "Skip" and "Save note" close the sheet (slides down, 200ms ease-in). No swipe-to-dismiss. No backdrop click to dismiss (D-08: explicit dismiss only).

### Screen 3: Expense History (`/expenses`)

**Layout:** Single column, `max-w-sm`, `px-4 py-8`. Mirrors home page list component.

**Back navigation:** "← Back" link at top, 14px, `--color-muted`. Navigates to `/`.

**Page title:** "All expenses" — 20px semibold, `--color-foreground`, `mb-4`.

**List:** Same visual treatment as today list on home screen (reuse the same Svelte component or identical markup). Group expenses by date with a date header row: date label (14px, `--color-muted`, e.g. "26 Apr") above each date group. Most recent first.

**Empty state:** Centered card (same `--color-surface` rounded card pattern). Heading: "No expenses yet". Body: "Log your first expense from the home screen." (14px, `--color-muted`). Include a "← Go home" link (14px, `--color-accent`).

**"View all history →" equivalent:** No need — this IS the history page.

### Screen 4: Edit Expense (`/expenses/[id]/edit`)

**Layout:** Single column, `max-w-sm`, `px-4 py-8`.

**Back navigation:** "← Back" text link, 14px, `--color-muted`, returns to referring page (home or `/expenses`).

**Page title:** "Edit expense" — 20px semibold, `--color-foreground`, `mb-6`.

**Form fields (top to bottom), each with `mb-4` separation:**

1. **Amount field:**
   - Label: "Amount" (14px, `--color-muted`, `mb-1`).
   - Input: numeric text input, `inputmode="numeric"`, full width, `min-h-[48px]`, border 1px `--color-surface`, border-radius 8px, `px-3`, 16px regular. Shows current amount as IDR-formatted string (e.g. "54000" stored → displayed as "54.000"). On blur: re-formats display.
   - Note: The system keyboard opens for this field — acceptable in the edit context (not the quick-add flow).

2. **Category field:**
   - Label: "Category" (14px, `--color-muted`, `mb-1`).
   - Rendered as a 3-column category tile grid (identical visual spec to home screen category grid). Selected category tile: border `2px solid var(--color-accent)`, background `#D4C9B8`. No "save on tap" behavior in the edit context — tile tap only selects, does not submit.

3. **Note field:**
   - Label: "Note (optional)" (14px, `--color-muted`, `mb-1`).
   - Textarea: full width, `min-h-[80px]`, border 1px `--color-surface`, border-radius 8px, `px-3 py-2`, 16px regular, resize: none. Placeholder: "Add a note...".

4. **Date field:**
   - Label: "Date" (14px, `--color-muted`, `mb-1`).
   - Input type="date", full width, `min-h-[48px]`, border 1px `--color-surface`, border-radius 8px, `px-3`. Displays in `id-ID` locale format visually but uses `YYYY-MM-DD` for value binding.

**Actions (below form, `mt-6`, `flex flex-col gap-3`):**

1. "Save changes" — primary, background `--color-accent`, text `--color-accent-foreground`, `min-h-[48px]`, border-radius 8px, 16px semibold, full width.
2. "Delete expense" — destructive, background `--color-destructive`, text `--color-destructive-foreground`, `min-h-[48px]`, border-radius 8px, 16px semibold, full width.

**Delete confirmation:** Inline — "Delete expense" button changes to a two-step confirmation state (D-06 context: simple confirmation, no modal):
- Step 1: Tap "Delete expense" → button text changes to "Tap again to confirm delete", border turns `--color-destructive`, background transparent, text `--color-destructive`. Second tap submits the soft delete (sets `is_deleted: true`).
- Confirmation state auto-resets after 3 seconds if not confirmed.

### Screen 5: Gear Menu (dropdown from home screen top-right)

**Trigger:** Tap gear icon (⚙) in top-right of home screen.

**Animation:** Dropdown slides down from the top-right over 150ms, `transform: scaleY(0)` origin top-right → `scaleY(1)`. Opacity 0 → 1 simultaneously.

**Positioning:** `position: absolute`, top aligned below gear icon (8px gap), right-aligned with page edge (matching `px-4`). Width: 200px.

**Dropdown style:** Background `--color-bg`, border-radius 12px, border 1px `--color-surface`, `box-shadow: 0 4px 16px rgba(0,0,0,0.10)`, `overflow: hidden`.

**Menu items (top to bottom, each `min-h-[48px]`):**
1. "Invite member" — navigates to `/settings/invite`. 16px regular, `--color-foreground`. Left icon: none (text only).
2. "Household details" — navigates to `/household`. 16px regular, `--color-foreground`.
3. Divider: 1px `--color-surface`.
4. "Log out" — submits POST to `/logout`. 16px regular, `--color-destructive`.

Each item: `px-4`, full width, flex row `items-center`.

**Dismiss:** Tap anywhere outside the dropdown. Implement via a transparent full-screen backdrop `div` at z-index below the dropdown, above the page content. `pointer-events: all`. Tapping it closes the dropdown with reverse animation (150ms).

**Backdrop:** No visible color (fully transparent). Z-index: backdrop at 40, dropdown at 50.

## Interaction Contract

### Numpad key press sequence
1. User taps a digit key → digit appended to internal amount string → amount display re-renders with IDR dot-separator formatting → no delay.
2. Leading zeros collapsed: "007" displays as "7".
3. Maximum amount: 99.999.999 (Rp 99 juta). If input would exceed this, ignore the tap (no visual error — just do nothing).
4. Backspace: removes last character. If string becomes empty → display resets to "0" in `--color-muted`.
5. "000" key: appends "000" to current string, subject to max-amount rule.
6. Amount display color: `--color-muted` when value is 0 (no digits entered), `--color-foreground` when non-zero.

### Category tile tap sequence (INPUT-05, INPUT-06)
1. User taps category tile → immediate visual pressed state (accent border + darker bg).
2. Debounce lock engaged (500ms window). During this window: `pointer-events: none` on all category tiles.
3. `client_id` UUID (generated at numpad-open per INPUT-07) used for the insert.
4. Expense saved to Supabase (optimistically update today list immediately).
5. Numpad resets to 0, amount display resets to "0" in `--color-muted` (D-04 — no animation delay on numpad reset).
6. Note bottom sheet slides up (D-07 — immediately after save, 250ms slide animation).
7. New `client_id` UUID generated for the next expense.

### Bottom sheet dismiss sequence
- "Skip" tap → sheet slides down (200ms) → user back on fresh numpad.
- "Save note" tap → note saved (PATCH to expense record) → sheet slides down (200ms) → user back on fresh numpad.
- No auto-dismiss. No swipe gesture (D-08).

### Expense row tap
- Tapping anywhere on a list row navigates to `/expenses/[id]/edit` (D-06).
- No swipe-to-delete on list rows (deferred — not in Phase 2 scope).

## IDR Formatting Contract

- Storage: integer (no decimals). Rp 54.000 stored as `54000`.
- Display in numpad: dot separator, no currency symbol. "54.000" not "Rp 54.000" and not "54,000".
- Display in lists: "Rp " prefix + dot separator. "Rp 54.000".
- Display in edit form amount field: dot separator, no "Rp" prefix (field label provides context).
- Locale: `id-ID` — dot as thousands separator. No decimal part (IDR has no sub-rupiah amounts).
- Format function: `amount.toLocaleString('id-ID')` produces correct output for integers.
- The `formatAmount` helper in the current home page (k/jt notation) is for the Phase 1 household overview only. Phase 2 list rows use full IDR formatting (`toLocaleString('id-ID')`) — no k/jt abbreviation in the expense list.

## Accessibility Notes

- All `<button>` elements (numpad keys, category tiles, gear icon) must have either visible text or `aria-label`.
- Backspace key: `aria-label="Delete last digit"`.
- Gear icon: `aria-label="Open menu"`.
- Category tiles: button text is emoji + label (visible); `aria-label` not required but emoji should be wrapped in `<span aria-hidden="true">` with label in `<span>`.
- Bottom sheet: `role="dialog"` with `aria-label="Add a note"`. Focus trapped inside while open.
- Delete confirmation inline state: `aria-live="polite"` on the button text region so screen readers announce the confirmation prompt.
- Touch targets: all interactive elements meet 44px minimum (WCAG 2.5.5). Numpad keys and category tiles exceed this (56px and 64px respectively).
