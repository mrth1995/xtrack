<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser storage -> Supabase auth client | Untrusted browser storage provides tokens to the client SDK. |
| public env -> Supabase client/server helpers | Developer-provided env values can point the app at wrong hosts or placeholders. |
| Supabase -> SvelteKit loader | Database/RLS errors cross into server-rendered app routes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08-01 | Spoofing | `indexedDbSessionStorage` | mitigate | Store only Supabase SDK session payloads under the SDK auth storage key; session validity remains checked by Supabase `getSession`/`refreshSession`. |
| T-01-08-02 | Information Disclosure | `validateSupabasePublicEnv` | mitigate | Reject placeholder/empty public env values before any network request logs DNS/fetch failures against placeholder hosts. |
| T-01-08-03 | Repudiation | `/logout` UI | mitigate | Visible POST logout form triggers existing `/logout` endpoint so sign-out is auditable through route execution. |
| T-01-08-04 | Tampering | app loaders | mitigate | Throw explicit 503 on Supabase errors; do not convert failed protected reads into empty trusted UI state. |
</threat_model>
