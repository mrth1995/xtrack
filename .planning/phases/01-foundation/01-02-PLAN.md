<threat_model>
- Threat: any authenticated user can read another household's rows if RLS is omitted or incomplete.
  Mitigation: enable RLS on all Phase 1 tables in the same migration and add tests for cross-household denial.
- Threat: invite codes can be reused or won by a race between two joiners.
  Mitigation: add a single SQL function that atomically validates, claims, and invalidates invites.
- Threat: plans pass locally while the database itself is still stale.
  Mitigation: include a blocking `supabase db push` task before final verification.
</threat_model>
