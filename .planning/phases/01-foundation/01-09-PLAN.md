## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| auth.uid() -> SQL helpers | Supabase authenticated identity enters SECURITY DEFINER functions. |
| client RPC -> invite functions | Untrusted invite codes and household IDs cross into database functions. |
| RLS policy -> table reads/writes | Database policies enforce household tenant boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09-01 | Elevation of Privilege | `public.is_household_member` | mitigate | Helper checks `public.household_members` with explicit `p_household_id` and `p_user_id`; policies call helper with fully qualified row columns. |
| T-01-09-02 | Tampering | invite acceptance/reuse RPCs | mitigate | `accept_household_invite` keeps `FOR UPDATE`; `get_or_create_active_household_invite` reuses active rows before insert. |
| T-01-09-03 | Information Disclosure | invite lookup | mitigate | `lookup_household_invite` only returns household confirmation metadata and maps invalid/expired/used states through stable hints. |
| T-01-09-04 | Denial of Service | recursive RLS | mitigate | Remove self-referential household_members policy subqueries that can recurse or misfilter. |
</threat_model>
