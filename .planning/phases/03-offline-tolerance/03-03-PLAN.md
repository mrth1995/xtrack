<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> IndexedDB | User expense data is written to persistent local storage. |
| IndexedDB -> network flush | Tampered local payload may be sent to the server. |
| lifecycle events -> flush | Online/visibility events can cause repeated retry attempts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | IndexedDB sensitive data | mitigate | `QueuedExpense` stores only current expense payload and retry metadata; no session, JWT, refresh token, or service-role key fields exist. |
| T-03-02 | Tampering/Elevation | Tampered queued payload household mismatch | mitigate | `flushExpenseQueue` compares queued `household_id` to current `householdId` and marks mismatch as `failed`; server plan uses locals as authority. |
| T-03-03 | Tampering/Repudiation | Replay/idempotency | transfer | Backend RPC plan enforces `ON CONFLICT (client_id) DO NOTHING`; this plan preserves stable `client_id` during retries. |
| T-03-04 | Spoofing | Expired auth | mitigate | `flushExpenseQueue` returns `paused: true` when session is not ready and marks auth failures as local `failed` rows. |
| T-03-05 | Denial of Service | RLS denial retry loops | mitigate | `classifySyncError` consumes `syncErrorCode: 'auth'`, `syncErrorCode: 'household_access'`, and `syncErrorCode: 'household_mismatch'`; these rows are marked failed and are not reattempted inside the same flush loop, while network failures return to queued. |
| T-03-06 | Elevation | Database migration/RPC access control | transfer | Backend migration plan owns RPC grants and server-side access checks. |
</threat_model>
