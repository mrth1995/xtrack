<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser IndexedDB | Expense data persists on the device outside the server trust boundary. |
| browser queue -> app server/Supabase | Queued payloads can be tampered before sync. |
| app server -> database RPC | RPC must enforce caller identity, household membership, and idempotency. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | IndexedDB queued expenses | mitigate | Tests must assert queued records contain only amount, category, note, spent_at, client_id, household_id, status, timestamps, and retry metadata; no auth tokens or service keys. |
| T-03-02 | Tampering/Elevation | Tampered queued payload household mismatch | mitigate | Tests must cover household mismatch classification as `failed`; later server work must use `locals.householdId`, never queued household_id, as authority. |
| T-03-03 | Tampering/Repudiation | Replay/idempotency | mitigate | Browser acceptance test must assert retry with the same `client_id` does not duplicate; backend plan adds `ON CONFLICT (client_id) DO NOTHING`. |
| T-03-04 | Spoofing | Expired auth | mitigate | Sync tests must verify flushing pauses or fails recoverably without a valid session. |
| T-03-05 | Denial of Service | RLS denial retry loops | mitigate | Sync tests must verify auth/RLS denial is classified as `failed` and not retried in a hot loop. |
| T-03-06 | Elevation | Migration/RPC access control | transfer | Backend migration plan owns RPC grants, `SECURITY DEFINER`, `SET search_path = public`, and RLS-compatible authority checks. |
</threat_model>
