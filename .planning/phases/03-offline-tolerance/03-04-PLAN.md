<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI -> IndexedDB | User actions create and mutate local records. |
| IndexedDB -> visible list | Local records are rendered alongside server rows. |
| browser lifecycle -> sync engine | Events trigger network writes. |
| local edit/delete -> server state | Never-synced rows must not create server tombstones. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | IndexedDB sensitive data | mitigate | UI writes only allowed `QueuedExpense` fields; no session/auth fields are passed to queue APIs. |
| T-03-02 | Tampering/Elevation | Tampered queued payload household mismatch | mitigate | UI stores household id for mismatch detection; server/RPC remains authoritative; sync module marks current-household mismatch as failed. |
| T-03-03 | Tampering/Repudiation | Replay/idempotency | mitigate | UI preserves stable `client_id` for queued retries and dedupes rendered rows by `client_id`. |
| T-03-04 | Spoofing | Expired auth | mitigate | Layout installs flush only after session-ready state; expired auth keeps rows local as failed/paused instead of dropping them. |
| T-03-05 | Denial of Service | RLS denial retry loops | mitigate | Failed rows expose manual retry and normal triggers but do not run a tight retry loop; syncing rows are locked during in-flight writes. |
| T-03-06 | Elevation | Database migration/RPC access control | transfer | Backend plan owns RPC access control; UI plan must not add service-role calls or bypass RLS. |
</threat_model>
