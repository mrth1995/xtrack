<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser IndexedDB | Expense payloads may be exposed or modified on device. |
| browser/form -> server action | Untrusted queued/form payload crosses into SvelteKit action. |
| server action -> Supabase RPC | Authenticated database call must enforce identity and household access. |
| migration -> production database | New RPC grant affects who can insert expenses. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | IndexedDB sensitive data | transfer | Queue storage plan limits local fields; this backend plan must not add auth/session data to RPC payloads. |
| T-03-02 | Tampering/Elevation | Tampered queued payload household mismatch | mitigate | `saveExpense` passes `p_household_id: householdId` from `locals`; SQL verifies `public.is_household_member(p_household_id, auth.uid())`. |
| T-03-03 | Tampering/Repudiation | Replay/idempotency | mitigate | SQL uses `ON CONFLICT (client_id) DO NOTHING` and returns the existing row for the same household/client_id. |
| T-03-04 | Spoofing | Expired auth | mitigate | RPC rejects `auth.uid() IS NULL` with SQLSTATE `28000`; server action still redirects unauthenticated callers before RPC. |
| T-03-05 | Denial of Service | RLS denial retry loops | transfer | Sync plan classifies access-denied responses as failed rows; this plan provides distinguishable access-denied RPC failure. |
| T-03-06 | Elevation | Database migration/RPC access control | mitigate | Migration uses `SECURITY DEFINER`, `SET search_path = public`, revokes from `PUBLIC`, grants only to `authenticated`, and checks household membership. |
</threat_model>
